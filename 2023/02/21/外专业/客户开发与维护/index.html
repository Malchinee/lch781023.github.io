客户开发与维护

绪言

市场营销——在人工智能时代还是不容易替代的，与人打交道的脑力劳动。但是但是，

  • 销售岗位与不同人打交道、解决的相关问题和产业工人调试机器并没有什么区别，熟练后并不难。

  • 客户关系一旦建立，后续的投入会少很多

问题：客户能给企业带来什么价值？

  1. 生存基础

    客户能带来收入，企业才有利润

  2. 公司发展的动力

    客户“买”，表示产业链的后端环节，离消费者比较近，容易知道消费者的需求、期望、痛点，可以对产品改进、研发提供方向

    技术本身无价值，只有有人运用它才有价值。认识是做的天花板

  3. 客户是最好的广告效应

  4. 市场竞争力

    占领客户如攻城略地，别人想要这批客户相比你守住客户更难

  5. 社会价值认同

    企业慈善、提高员工福利等，人权认证（员工基本生活得到保证），如禁止企业雇佣童工等。公司的价值得到社会的承认

客户开发的挑战：

  • 客户在哪里

    到处碰壁

  • 如何与客户建立联系

    胆怯、不知所措

  • 如何与客户保持长期关系

    不善打交道

  • 如何提成业绩，控制成本

    只知道请客、送礼、求人。请客、送礼这些适可而止

所以怎么办呢？

1.寻找潜在客户

客户开发的核心是建立信任

  • 潜在客户线索，可能称为潜在客户的个人或企业机构的名字，或相关的联系方式。线索一旦通过了验证，就成了潜在客户

    线索的来源：朋友和熟人、无穷的关系链、有影响的人物、其他组织的人员名单、电话黄页、“休眠”的客户、陌生拜访、观察、网络社区、报刊、广告、讨论会、商业展览、销售线索俱乐部、无竞争关系的其他销售人员

潜在客户可能购买你的产品或服务的个人或公司。潜在客户线索一旦通过了验证，就成了潜在客户

潜在客户发掘的方式：

  • 主动式：电信访问、陌生拜访（不推荐）B2B

  • 被动式：抛出诱饵、开门侯客.B2C，有效且成本低

另外，潜在客户还应该：

  • 有购买某种产品或服务的需要

  • 有购买能力

  • 有购买决策权

成为客户之后，非常重要的一步是建立客户档案

潜在客户档案主要内容：

  • 客户信息：公司名称、地址，主营业务内容与状况，主要联系人姓名、职务、联系方式、兴趣爱好、家庭状况等

    掌握以上信息后，可以抓住一些细节维护与客户的关系，如兴趣爱好，可以增进与客户的联系

  • 拜访记录：记录该客户的所有相关信息，每次接触的方式、时间、地点、费用和内容

  • 交易记录：交易产生后的相关记录。货物、货款、应收账款等

所以，优秀的记录，有助于分析客户状况，掌握销售进度，提高服务水平

2.初次会晤

不断提升概率，找到最有价值线索的概率

不能没有预约就往前往，应该先预约

电话预约六步：

介绍细节的危害:

  • 问的再多，客户都不可能通过一个电话成交
  • 从人接受的信息的心理来看，电话介绍的产品信息不全，并且随着谈话的进行，前面的信息都会忘掉

切忌介绍细节，都会影响成功的概率

  1. 打电话之前的计划与组织

    先要考虑好如何将客户拿下，提前组织好语言，文案。开始时怎么介绍，谈话过程中如何吸引注意力，下一步怎么推进等等，整理发言稿。极端方式：决策树

  2. 介绍你自己和你所在的公司

    不要报全名，可以说公司的xxx经理等职位。此时风险最大，语速要快一些，应立马进行下一步

  3. 导入主题

    不同行业主题不同，不要介绍产品本身，介绍好处利益，可以适当夸大一点吸引。

  4. 激发兴趣的陈述

  5. 说明打电话的目的并请求约见

  6. 处理拒绝

处理拒接：

定义：任何潜在客户所给出的无法进行会面的理由，都可以视为拒绝

在遇到客户的拒绝借口时，不可绝望的挂掉电话，挂掉电话意味着以前的努力白费

客户拒绝的理由：

  1. 资金紧张：采购批量小，多批次，需要吃透公司的各种销售政策。
  2. 没有时间，准备出差：另外定时间
  3. 有需要的话我跟你联系：（礼貌的拒绝），可以给客户一些资料、个人联系方式
  4. 对原供应商比较满意：“我们以前客户也是这么说的，但是见到我们之后xxxxxx”
  5. 没兴趣，没有必要
  6. 先传一份资料吧：将资料发送给客户，发送后确认客户是否收到
  7. 我需要和其他人商量：“为了你能跟别人介绍更准确，发您一份资料”

初次会晤：

  • 做好见面前的准备：查询公司的具体数据，业务、竞争力等；见面准备聊什么事情，第一次见面也不可能会达成交易
  • 初次会晤：
    • 确认对方，问候
    • 自我介绍
    • 微笑、握手
    • 递上名片
    • 开场寒暄
    • 切入正题

初次会晤的核心任务就是确认顾客是否有需求，或者尝试激发顾客需求

3.识别问题

帮客户找出一些公司存在的痛点，与自己的业务结合，和客户聊一聊，推销自己的业务

传统销售沟通方式：

销售人员直接向潜在客户介绍产品或服务，告诉他们为什么要购买。设定前提是，一个全面、成功的表述，会自然而然地达成交易

缺点：把购买当成一个可以被摆布和强迫地对象。不能保证客户认识到存在的问题，以及购买产品地必要性

讨论提问式销售沟通方式：

以提出问题，聆听潜在客户回答的方式，引导并鼓励其意识到自身面临的问题和需求，在此过程中，销售人员帮助潜在客户得出结论

要求：销售人员有较高的提问和聆听的技术

提问的过程：

  1. 与潜在客户一起识别现状
  2. 确认潜在客户追求的理想状态
  3. 继续或改变交流内容
  4. 检验是否理解客户的意思
  5. 确定问题

另外要记录笔记，防止遗忘，表达对客户的尊重

常用的提问技巧：

  • 使问题简单易答
  • 自己承担责任
  • 给提问留下空间
  • 询问利益、期望而非产品特性

4.现场演示

客户购买的不是产品和服务本身，而是解决问题的方法

眼见为实，成功的现场演示能够大大增强客户对公司的信赖感

如何说服客户：

  • 感觉自己“内向”，不知道“该说些什么”，或者“怎么说才能打动客户”

  • 客户的问题难以预料，自己的现场应变能力不足，无法掌控局面

  • 谈话主题漂移，找不清楚主题

做法：

  1. 分析客户的心理过程

  2. 应用：“卖点——利益”综合分析表

    对公司提供物机器给客户带来的利益进行详尽分析，并精心准备展示方法

    形象化展示的思路：

    • 展示样品
    • 免费测试/试用
    • 与其他产品的比较
    • 使用视听设备/资料
    • 他人推荐
    • 显示投资回报
    • 提供保证书或认证证书
    • 到公司参观考察

实践的要点：

  • 建立个性化“卖点——利益”综合分析表
  • 事前多练习，事后多优化
  • 习惯成自然

总结：核心是把组织的提供物转化成客户利益

  1. 根据客户的性格特征调整演示方法

  2. 产品示范的计划

    1. 事先做好示范计划
    2. 考虑产品示范的环境
    3. 逐步考虑，并适当提问，确保客户理解产品优势和特征
    4. 鼓励客户参与示范的每一个步骤
    5. 针对客户具体需求

  3. 几个注意要点

    • 从销售人员第一次与客户会面开始，平均需要五次拜访才能达成一项交易，不要操之过急
    • 每次拜访前可以准备多个目标，然后根据现场情况进行调整，尽量使每次拜访都能产生价值
    • 单次拜访即将结束，都应当跟客户进行简单的回顾，并为下次拜访确定大致的议题，做好备忘录

5.异议处理

异议：在人员推销过程中，对相关信息或销售人员的要求表示疑问、反对或其他抵触的行为

异议产生的原因：

  • 交易本身：
    • 需要提供更多的信息
    • 需要更好的条件
    • 对交易条款的利弊心存疑虑
    • 对交易的履行还不放心
    • 购买时机是否恰当
  • 公司：
    • 公司信誉问题
    • 产品与相关服务问题
    • 公司相关保障能力问题
    • 销售人员问题
    • 企业服务与宣传方面的问题
  • 客户：
    • 客户未发现问题与需求
    • 客户未意识到改进现状的新方法
    • 客户的消费偏见与习惯
    • 客户无力支付
    • 客户无权购买
    • 客户有固定的采购关系
    • 客户的偶然因素，如一时心情不好
    • 难以接受变化
    • 周围人的压力

应对异议的时机选择：

  • 预先处理客户异议

    能做的也只有这条，实现把这些做好

  • 立即处理客户异议

  • 推迟处理客户异议

  • 不处理客户异议

应对异议的方法：

  • 预先处理

  • 正面否定

  • 间接否定

  • 优劣弥补

    突出价格优势、送货周期等

  • 附和转移

  • 自食其果

    “你们公司是小公司”

  • 举证

6.销售谈判

非核心业务，不直接用于生产的货物谈判简单一些

步骤：

  • 谈判前期准备

    • 明确谈判的议题：一个或多个

    • 明确谈判的目的和目标：针对不同的议题，确定不同的目的和目标，并且明确最高目标、中级目标和最低目标

    • 明确自身提供的价值、筹码与要求，务求详尽

    • 制定谈判计划：谈判地点与实践、谈判期限、活动安排、过程安排、人事安排、各环节的时间掌控

    • 对谈判对方资质及谈判人员的详细调查

    • 对方筹码与要求、谈判计划等的评估

  • 人员组织

    针对谈判议题、我方谈判目的和目标、谈判地位，对方谈判人员情况等，确定我方谈判人员。也就是说，谈判人员的选择要考虑到团队的专业知识构成、职位级别、权限、谈判风格与技术等

    要对谈判的简单方面首先达成共识

    对于重要、复杂的谈判，还需要建立后方谈判支持团队，为前线的谈判人员提供信息和决策支持

  • 谈判过程

    谈判开始前，要尽量争取一个对自身有利的谈判框架，把握主动

    谈判内容的次序安排同样重要，一般先谈容易达成公示的，然后逐步缩小分歧

    恰到好处的接待、娱乐休闲活动、阶段性成果庆祝会、非正式接触有助于消除敌意、调节谈判气氛与节奏、促成谈判双方的理解与信赖，达成谈判目标

    谈判过程除了要善于说，更要善于听，尽快把握对方的要求和意图

  • 谈判签约

  • 后期落实

7.交易促成

一、购买决断时的危机感：

客户的顾虑：

  • 担心没有考虑周全

  • 担心未来可能的损失

  • 担心亲人或朋友、公司同事或领导不满意

  • 纯粹的犹豫不决

销售人员的顾虑：

  • 担心被拒绝

    将“促成购买决定”视为帮助客户的机会；一旦被拒绝，客户所拒绝的不一定是销售人员个人，而可能是对产品或服务，或者购买后的后果存在顾虑

二、促成购买决定的时机：

  • 基本要点：只有当产品演示、异议处理完成后，才能实施“促成购买决定”行为

  • “感兴趣 + 异议处理完毕”后，语言肯定、肢体语言显示、和谐沉默等信号出现时，常常是有利的时机

  • 在正常的销售洽谈过程中，存在多个时点适于销售人员启动“促成购买决定”

如果没有交易，那就意味着什么也没发生

三、促成购买决定的技巧：

  • 直接促成

  • 综合利益

    销售人员将客户已表示有兴趣的一些买方利益综合起来，再次请客户注意，以促使其作出购买决定

    一般以客户最认同的利益作为开始，以客户曾提出异议的利益作为结束

  • 对比

    销售人员画出“T”型图，将购买理由放到T的一边，将不购买的理由放到另一边。制作对比表后，就应向客户逐一说明并邀请成交

  • 连续诱导肯定回答

    对一系列问题做习惯性的肯定回答，进而促使客户做出购买决定

  • 揣测型

    避开促成购买决定的主要问题，从一些枝节问题或后续问题入手，向用户提出选择性问题

四、告辞与回访

在成功签约后，销售人员应对客户的合作表示感谢。谢意的表达必须是真诚的，不掺杂任何屈尊或恩赐之意。销售人员应该让客户感到，客户随时都会受到认真的接待，交易的达成是值得庆贺的

五、后续工作

  • 完善更新客户数据库（客户相关信息，拜访记录等），注意小公司不要透露太多，个人可替代性就会增加

  • 与客户进行业务合作，履行承诺

  • 与客户进行感情联络，增进互信

  • 检测客户满意度

  • 探寻更大合作空间

8.关系推销

一、客户关系的发展阶段

  • 识别：寻找适合进行关系推销的对象

  • 探索：相互接触和进一步评估

  • 扩展：不断扩大合作的范围或领域

  • 承诺：基于信任与默契，不断维护巩固双方关系
  • 终止：减少或终止与对方的关系

与客户建立长期关系的好处

  • 向新客户进行推销的成本远高于向老客户推销的成本
  • 稳定销量，保障公司各方面的业务稳定
  • 便于提高合作效率，降低成本
  • 便于遏制竞争
  • 感知市场变化

二、影响长期关系发展的重要因素

  • 信任、承诺、合作
  • 共同目标
  • 相互依赖与实力
  • 绩效满意度
  • 适应性
  • 共享技术
  • 转换条件与壁垒
  • 结构性联系

9.客户信用管理

一、信用销售（赊账），是以信用为基础的销售。

信用销售的利弊

  • 对买方的利益
    • 缓解资金周转的压力
    • 给买方发现产品质量问题的时间，在付款问题上占据主动地位
    • 利用卖方资金可以减少自身资金压力，降低资金成本

  • 对卖方的利益

    • 刺激购买力
    • 提高卖方竞争力
    • 稳定客户
    • 减少企业库存

  • 对卖方的弊端

    • 资金压力
    • 增加成本：短缺成本、管理成本、机会成本和坏账成本
    • 影响现金流

信用管理：

  • 概念：授信者对信用交易进行科学管理，控制信用风险的专项管理
  • 全程信用管理：跟随销售和回款的交易环节，在事前中后，以客户和欠款为中心实时的监控
  • 信用政策：包括信用标准、信用额度、信用期限、现金折扣和收账政策。分为财务型和销售型两种

二、信用管理的职能

  1. 客户档案管理
    • 建立和维护易检索的客户档案
    • 及时提供公司内的客户信息服务
    • 动态更新客户信息
  2. 客户授信
    • 以分析模型量化赊账风险
    • 根据模型评分分制定信用成果
    • 审核合同和核准赊账
  3. 辅助市场开拓
    • 利用客户档案，扩大销售的机会
  4. 应收帐款管理
    • 诊断逾期应收账款
    • 制定追授策略和流程
    • 寻求法律解决追授
  5. 逾期账款追收
    • 动态跟踪客户欠款水平
    • 分析欠赊账款或欠赊成因
    • 掌握欠款总额的规模
    • 调控现金流量

应收账款管理：

  • 制定信用评定、审核制度

  • 在销售合同中明确各种交易条款

  • 强化客户的回款意识

  • 定期的财务对账

  • 制定合理的激励制度

  • 控制发货以减少应收账款

对销售人员的管理

  • 对产品铺货率的正确理解
  • 加强原则性教育
  • 适度的赊账、代销
  • 加强销售人员的回款意识
  • 制定科学，适当的账款回收考核机制
  • 加强销售人员的终端管理和维护能力
  • 提高销售人员的追款技巧
  • 销售人员要注意终端经营不善的危险信号

应收账款处理方法

  • 正确处理方式：
    • 建立形象、明确政策
    • 销售资料齐全
    • 完善客户跟进制度
    • 定期探访
    • 技巧训练
  • 被拖欠款项的处理方法
    • 检查销售文件是否齐备
    • 建立账款催收制度
    • 让客户了解最后期限的含义及结果
    • 行动升级
    • 假起诉
  • 对于呆账、死帐的处理
    • 折让
    • 收回货物
    • 处理抵押品
    • 寻求法律协助
    • 诉讼保全
